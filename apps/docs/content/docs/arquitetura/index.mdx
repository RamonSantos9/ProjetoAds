Ao adotar uma arquitetura de **Monorepo**, o PodcastAds garante um ecossistema coeso onde o núcleo lógico, a interface visual e o motor de documentação coexistem de forma harmoniosa, maximizando o reuso de código e a velocidade de entrega.

## Estrutura de Monorepo 🏗️

Utilizamos o **[Turborepo](https://turbo.build/)** para orquestrar nossos espaços de trabalho (_workspaces_) via `pnpm`. Esta abordagem permite o cache inteligente de builds e testes, reduzindo drasticamente o tempo de CI/CD.

```mermaid
graph TD
    Root[Monorepo Root]
    Apps(apps/)
    Pkgs(packages/)

    Root --> Apps
    Root --> Pkgs

    Apps --> Docs[docs - Portal & Admin]

    Pkgs --> UI[@xispedocs/ui - Design System]
    Pkgs --> Core[@xispedocs/core - Logic & Types]
    Pkgs --> MDX[@xispedocs/mdx - Content Engine]

    Docs -.-> UI
    Docs -.-> Core
    Docs -.-> MDX

    UI -.-> Core
```

### Por que Turborepo?

Para suportar iterações em larga escala, o Turbo coordena a execução de scripts. Quando alteramos um componente no pacote `@xispedocs/ui`, apenas as aplicações que dependem dele são invalidadas, garantindo que o ambiente de desenvolvimento permaneça ágil (HMR sub-segundo).

## O Ecossistema App Router

Nossa base tecnológica é sustentada pelo **Next.js >= 15** em sua topologia `App Router`, permitindo:

1. **Server Components por Padrão**: Minimizando o bundle de JavaScript enviado ao cliente e otimizando o SEO.
2. **Layouts Aninhados**: Garantindo que a Área de Administração (`/admin`) compartilhe a mesma casca estrutural que o Portal do Ouvinte, mantendo a consistência visual.
3. **TailwindCSS v4**: Estilização atômica e performática, abstraída em nosso pacote `@xispedocs/ui` e exportada via variáveis CSS modernas.

Esta união de tecnologias permite uma plataforma que escala de simples apresentações acadêmicas a sistemas de gestão de mídia complexos com esforço mínimo de manutenção.
